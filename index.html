ESP32 Monitor — the desktop app

Complete & defended · firmware + app + hardware

The system has three parts: ESP32 sensor nodes that measure and send data over Bluetooth; a cross-platform desktop app (Windows & macOS) for live values, graphs and cloud-synced storage; and the Sensirion MyAmbience mobile app for a quick phone view. I built every layer — the C++ firmware (~1,700 lines), the Python app (~9,700 lines), the 3D-printed hardware, and the data analysis.

My contribution: most hobby IoT devices hard-code the trade-off between live data and battery life. Mine makes it a runtime setting — three power modes taking current from ~64–93 mA down to ~4–7 mA, extending battery life from days to weeks. It logs locally and to the cloud with crash-safe auto-resume, hit 100% BLE reconnection and zero packet loss up to 5 m, validated over 7–22-day field runs with statistics in MATLAB and current measured on a Rohde & Schwarz oscilloscope.

ESP32_DevKit.ino — my firmware
// Clean radio teardown before deep sleep — cutting BLE/Wi-Fi first
// drops the LDO's residual 60-80 mA load. From my ESP32 firmware.
void enterDeepSleep(uint32_t seconds) {
  saveState();                        // persist epoch + config to NVS
  if (g_nimbleInited) {
    NimBLEDevice::getAdvertising()->stop();
    NimBLEDevice::deinit(true);      // release the BLE controller
  }
  WiFi.mode(WIFI_OFF);
  esp_bt_controller_disable();
  delay(300);                       // let the 3.3 V rail recover
  esp_sleep_enable_timer_wakeup((uint64_t)seconds * 1000000ULL);
  esp_deep_sleep_start();             // ~64 mA -> ~6.9 mA (~89% less)
}
